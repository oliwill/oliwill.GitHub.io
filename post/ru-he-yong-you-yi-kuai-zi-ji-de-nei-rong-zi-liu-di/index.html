——使用 Gridea 搭建静态 blog

2020 年，因为一些原因，使用了 10 年的微信账号被变成了一个「私域」账号。那期间，真实感受了什么是数字幽灵，我可以听到人群的声音，可以听到人群中某些人在呼唤我，但是我没法直接反馈给他们。
具体的表现是，一对一聊天时，我发送任何消息都只会得到一个提醒，提醒我账号违规无法发送消息。当然别人给我发消息也是被告知一样的情况。
而在群里的对话，我可以阅读，也可以被群友@，但是我依旧没法发送任何消息。

通过这件事我才更深刻地意识到人与人如果只靠一个 app 链接是一场灾难，而如果一个人把生活也只展示在这一款 app 那真的随时都有灭顶之灾的危险。因此，我决定重新搭建一个 blog，通过 blog 来相对自由地记录自己的生活。

而对 blog 的选择，首先要确保所有权和使用权都属于我，这样才能避免之前的遭遇。其次，希望它搭建简单，经济实惠，又兼具一定美观性。综合了 Wordpress、Ghost、Typlog 等之后，最终选择了 Gridea。

##搭建过程
###安装

  1. 通过 Gridea 官网下载安装
    ###配置

Gridea 提供了将博客部署到 Github Pages，Coding Pages 和通过 SFTP 的方式上传到自己的服务器的功能。

  1. 创建 GitHub 账号，并创建一个仓库 Token。
    1. 点进 GitHub，按提示进行注册并登陆。
    2. 点击创建一个 token，勾选 repo 的相关权限。
    3. 生成后的 token 一定要记录下来并妥善保管，因为这个页面一旦关闭后就无法再次看到。
  2. 创建仓库，存放构建后的网站文件。
    1. 创建一个公开仓库，名为用户名.github.io
  3. 进入 Gridea 进行远程设置。
    1. 按照下图的示例进行设置，然后点击「检测远程链接」，进行配置检查。
  4. 同步。
    1. 点击 Gridea 中的同步按钮即可。
      完成以上几步后一个静态 Blog 就已经完成了初步搭建，如果还想折腾主题，评论，拥有自己的域名的话后续会继续更新。
